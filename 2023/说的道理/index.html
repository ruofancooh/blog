如何应对拖延

先写最重要的部分，不要迷失在语言表达，想梗和想标题上。

有想写的马上写，可以先不发，或者先发一部分，后来再拼起来。

有想做的马上做，可以先做一部分。

有时候脑子里总在想一个标题，其实自己根本对要写的内容都没有明确的理解，只是觉得这个标题特别吊，光把时间用到“预想写完之后的结果”了。

从阮一峰的一篇文章里看到，博客需要先写快，后写好。

从B 站的一篇笔记看到，满足欲望不是什么坏事。在没有特别重要的事情的情况下，先把最想做的做了。

睡一觉起来发现海港没了。

常问 GPT

由于我经常对一个问题，一会想用这种解决方法，一会想用那种解决方法，结果想来想去都是把以前有印象的碎片拼接起来，不会凭空创造出新的解决方法。

比如写代码，只是看到代码量增多，造成了没有意义的正反馈，实际结构写得非常混乱。

有那么多包不用，用 Python 就得调包。因为我总是想把问题以现有的经验解决。

到最后发现，本来是想解决一个小的问题，总是想把它做大来解决更多问题。假舟楫者非能水也而绝江河，我搁那终日而思去了。

我问 GPT：

py 有没有乐理相关的库？

它直接给了我几个名字：music21、mingus

这还用得着我去写吗？算法不会，调包不愿意，属于上也上不去，下也下不来。

然后我发现我的注意力又转移了：本来想学乐理，后来想边学乐理边练 python，后来全纠结 python 去了。

我不知道我对什么感兴趣。
