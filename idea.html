VSCode是有局限性的

发表于 2024-04-11 更新于 2024-07-21 分类于 牢骚系列

VSCode 它是一个轻型软件，是一个壳子。是一个代码编辑器，不是一个集成开发环境。

即使我们可以在它的插件市场里下载一个对 Python 语言提供支持的插件，这个插件可以满足你在编辑一个后缀为 .py 的文本文件时以不同颜色显示你的代码，自动补全你的代码；下载一个格式化代码的插件，一键让你的代码排版更好看。但是它没法运行你的代码，因为你还没有下载 Python 本体。Python 本体包括一些别人已经写好的，被无数人使用过的代码，这被称为模块、库或者包；还包括 Python 解释器和 Python 虚拟机。虚拟机是一个模拟的小电脑，解释器把你写的 Python 代码翻译成虚拟机能读懂的语言，在虚拟机上运行。如果你没有 Python 环境，它或许不能自动补全你的基础语法，笔者没试过。但如果你没写 import 语句，它肯定不能自动补全你想要的。

语言 必需品（加上任一个文本编辑器） 再往上（再再往上，就是 IDE）
C/C++/ASM MinGW CMake（没试过）
JavaScript 浏览器内核/ Node.js npm

到了 Java 这里，情况变得复杂了：首先你还是需要 Java 虚拟机和 Java 编译器，它们被打包在了一个名叫 JDK 的东西里。把 JDK 解压之后，你需要在 VSCode 的插件市场里下载一个 Java 扩展包，这个扩展包里有 6 个插件：有语言支持的、排错的、测试的、在 Github 上寻找相近代码例子的、工程管理的、与 Maven 交互的。其中 Maven 是一个管理 Java 工程的工具，你如果想使用它——对不起，你得自己下载。下载完之后还要修改 Maven 的配置文件，设置从哪个网络地址获取别人已经写好的包，存到电脑本地的哪个文件夹里。之后要修改 VSCode 的插件的配置文件 C:\Users\yourname\AppData\Roaming\Code\User\settings.json，把 Maven 本体在电脑上的路径写进去，要不然 VSCode 找不到它。还要把 JDK 在电脑上的路径写进去，要不然 VSCode 找不到它。你当然可以通过在命令行上敲命令来使用 Maven，但插件能帮你自动敲命令，所以这么一通配置，好像也不亏。所以折腾 VSCode 的过程就是折腾插件的过程，它里面甚至还有玩 4399 小游戏的插件：

你就这么得过且过地往 VSCode 上安装插件，直到有一天，你遇到一个名叫 JSP 的东西。JSP 就是让你把 Java 代码和网页混在一起写，然后运行在一个名叫 Tomcat 的东西上。你还是可以通过 Maven 插件来新建一个 JSP 项目。然后下载 Tomcat、下载一个叫 Community Server Connectors 的插件，对插件一通折腾，把 JSP 的代码复制进来，运行成功了，很好。但是在运行成功之前你就会发现：你找不到一个对 JSP 语言提供支持的插件，只有语法高亮的。它没法排你的错、没法给你补全代码。这时候你懊恼也没用，因为这插件是人家写的，让你写你也写不出来。你的矛盾就从如何写 JSP 代码上转移到了为什么我还要坚持使用 VSCode 上。然后你下载了 IDEA，发现最新的 IDEA 免费版也不对 JSP 提供支持，除非升级到付费版的。你上网一搜，发现有人说 JSP 已经过时了，现在流行前后端分离。你的矛盾又转移到了我为什么要上这门课，上大学到底是为了干什么，应该干什么这一件事上。

然后你就会从四个方面进行自我批评：

  1. 既然开这门课，肯定有它的道理。你这门课的知识都没弄明白，还能知道前后端分离是什么意思吗。
  2. 每学期都买书，每学期都不看。人家都把书出了，能力百分之一万比你强，你凭什么不看。
  3. 只要这个网站还存在一天，你的病就永远好不了。
  4. 现在 IDE 这么发达，你怎么不想想没有自动补全功能、没有检查语法功能的时候，代码是怎么写的。

从四个方面进行反向自我批评：

  1. 现在就在写一个前后端分离的系统。
  2. 你的动手能力和试错能力是有的，但是性子太急了。你看原理的时间比别人花得少，造成试错的时间比别人花得多。
  3. 这至少能让你有事干，你至少没有危害社会。
  4. 你回到石器时代去吧。

批评完之后你会有这几种选择：

  1. 继续坚持用 VSCode
  2. 在网上找 IDEA 的旧版本，试错
  3. 改用任课老师用的 Eclipse 安装包，试错
  4. 下载新版 Eclipse，试错
  5. 下载付费版 IDEA + 破解补丁，试错

你想要的只是一个代码自动补全、查错、格式化的功能。这个功能可能在未来会节省你的时间，但现在显然在拖你的时间。如果你现在不花时间搞到这个功能，未来可能花更多时间查错。从这一点就能看出，矛盾是无处不在的。你的最终目的是把实验报告交上去，得到这一门课的成绩，但你的本我和超我显然都比别人强大。所以为了不继续内耗，你就会形成一个自我保护机制，就是拖，等拖到最后时间紧迫了，你就不会想把实验报告写得那么好了，完成了就行。随之而来的是第三次自我批评：前几天还说论迹不论心，这几天又在论本我超我。你这个人对自己都是双重标准。你写这么多自我批评，无外乎就是为了给别人看，你这人就是在演戏，立这个人设，好像让别人觉得你这人每天都在纠结，对自我要求严格，但是什么事情都没做。其实你根本就没有病，你的所有病都是装出来的。包括你把“自己在演戏”这件事情说出来的本身，也是在演戏，就是为了给别人留下一种印象：这个小伙子挺有意思的，挺幽默的，幽默的人总不会干出什么极端的事，以后和他一起生火做饭的时候，出了什么问题，他至少不会把锅掀了。
