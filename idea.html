VSCode是有局限性的

发表于 2024-04-11 更新于 2024-04-12 分类于 音乐

VSCode 它是一个轻型软件，是一个壳子。是一个代码编辑器，不是一个集成开发环境。

即使我们可以在它的插件市场里下载一个对 Python 语言提供支持的插件，这个插件可以满足你在编辑一个后缀为 .py 的文本文件时以不同颜色显示你的代码，自动补全你的代码；下载一个格式化代码的插件，一键让你的代码排版更好看。但是它没法运行你的代码，因为你还没有下载 Python 本体。

Python 本体包括一些别人已经写好的，被无数人使用过的代码，这被称为模块、库或者包；还包括 Python 解释器和 Python 虚拟机。虚拟机是一个模拟的小电脑，解释器把你写的 Python 代码翻译成虚拟机能读懂的语言，在虚拟机上运行。

到了 Java 这里，情况变得复杂了：首先你还是需要 Java 虚拟机和 Java 编译器，它们被打包在了一个名叫 JDK 的东西里。把 JDK 解压之后，你需要在 VSCode 的插件市场里下载一个 Java 扩展包，这个扩展包里有 6 个插件：有语言支持的、排错的、测试的、在 Github 上寻找相近代码例子的、工程管理的、与 Maven 交互的。

其中 Maven 是一个管理 Java 工程的工具，你如果想使用它——对不起，你得自己下载。下载完之后还要修改 Maven 的配置文件，设置从哪个网络地址获取别人已经写好的包，存到电脑本地的哪个文件夹里。之后要修改 VSCode 的 Maven 插件的配置文件，把 Maven 本体在电脑上的路径写进去，要不然 VSCode 找不到它。还要修改 VSCode 的 Java 扩展的配置文件，把 JDK 在电脑上的路径写进去，要不然 VSCode 找不到它。

你当然可以通过在命令行上敲命令来使用 Maven，但插件能帮你自动敲命令，所以这么一通配置，好像也不亏。所以折腾 VSCode 的过程就是折腾插件的过程，它里面甚至还有玩 4399 小游戏的插件。

你就这么得过且过地往 VSCode 上安装插件，直到有一天，你遇到一个名叫 JSP 的东西。JSP 就是让你把 Java 代码和网页混在一起写，然后运行在一个名叫 Tomcat 的东西上。

你还是可以通过 Maven 插件来新建一个 JSP 项目。然后下载 Tomcat、下载一个叫 Runtime Server Protocol UI 的插件，对插件一通折腾，把 JSP 的代码复制进来，运行成功了，很好。但是在运行成功之前你就会发现，倒数第二个对 JSP 提供支持的插件已经停止维护了，就是用不成了。如果还能用，你现在看到的这一篇文字就不会被发出来。

只剩一个 JSP 语法高亮的插件，它没法排你的错、没法给你补全代码。这时候你懊恼也没用，因为这插件是人家写的，让你写你也写不出来。你的矛盾就从如何写 JSP 代码上转移到了为什么我还要坚持使用 VSCode 上。

然后你下载了 IDEA，发现最新的 IDEA 免费版好像也没办法格式化 JSP 代码。你上网一搜，发现有人说 JSP 已经过时了，现在流行前后端分离。你的矛盾又转移到了我为什么要上这门课，上大学到底是为了干什么，应该干什么这一件事上。
